# AI vs. Human Text Detection

Project 1 — Intro to LLMs. Detects whether a piece of text was written by a
human or generated by AI, using 6 trained classifiers and a Streamlit web app.

## What's in here

```
ai_human_detection_project/
├── app.py                      # Streamlit web app (Part 2)
├── requirements.txt
├── README.md
├── utils/
│   └── text_features.py        # shared linguistic-feature extractor (used by both notebook & app)
├── models/                      # trained model artifacts (see "Models" below)
├── data/
│   ├── training_data/train.csv
│   └── test_data/test.csv
```

## Models — what's already trained vs. what you need to run

This project was built in a sandboxed environment **with no internet access**,
so TensorFlow, gensim, and Streamlit could not be installed there. Here's the
honest breakdown:

**Already trained and saved (real results, run in the sandbox):**
- `models/tfidf_vectorizer.pkl`, `models/linguistic_scaler.pkl`
- `models/svm_model.pkl` — SVM, 98.0% accuracy
- `models/decision_tree_model.pkl` — Decision Tree, 91.5% accuracy
- `models/adaboost_model.pkl` — AdaBoost, 94.8% accuracy
- `models/fnn_sklearn_model.pkl` — a feedforward neural network (sklearn `MLPClassifier`), 99.1% accuracy — a fast, dependency-free FNN reference

**You need to run once locally** (after `pip install -r requirements.txt`) to generate:
- `models/embedding_model/word2vec.model` + `tokenizer.json` (Word2Vec embeddings)
- `models/fnn_model.h5`, `models/lstm_model.h5`, `models/cnn_model.h5` (Keras FNN/LSTM/CNN)

Run:
```bash
python scripts/03_deep_learning_RUN_LOCALLY.py
```
This took only a few minutes on a laptop CPU in testing; a GPU will be much faster.
It writes the three `.h5` models, the Word2Vec embeddings, training-curve plots,
and `deep_learning_results.json` (metrics you should paste into the notebook's
Section 4 comparison once you've run it — there's a clearly marked cell for this).

The notebook (`notebooks/project1_notebook.ipynb`) contains the full pipeline.
Cells using only `sklearn`/`pandas` already show their real output. Cells under
"Section 3: Deep Learning Models" require TensorFlow/gensim — run them locally
(or open the notebook in Google Colab) to execute and populate their outputs.

## Running the app

```bash
streamlit run app.py
```
The app auto-detects which models exist in `models/` and only lists those in
the model selector — so it works even before you've trained the deep learning
models (you'll just see SVM / Decision Tree / AdaBoost / FNN-sklearn until you
run `scripts/03_deep_learning_RUN_LOCALLY.py`).

## Dataset

`train_data_with_labels.xlsx` → 8,176 documents, perfectly balanced
(4,088 human / 4,088 AI), label 0 = human, label 1 = AI. Split 80/20 stratified
into `data/training_data/train.csv` and `data/test_data/test.csv`.

## A note on scope/limitations (worth saying in your demo video)

- The dataset's texts are long, formal/academic-style passages. The models
  generalize well to held-out test data **from the same distribution**, but
  accuracy will likely drop on very short, casual, or out-of-domain text —
  worth demonstrating and discussing in your video.
- Feature comparison (run on the full dataset): TF-IDF alone reached ~97.9%
  accuracy; linguistic features alone ~82.5%; combining them didn't improve
  on TF-IDF alone for the linear model — i.e. TF-IDF is already capturing
  most of the separable signal for this dataset. That's a real, useful
  finding for your Section 4 write-up, not a bug.
- "SVM" here is implemented as a calibrated `LinearSVC` (linear kernel),
  which is standard practice for high-dimensional sparse TF-IDF features and
  is dramatically faster than a kernel SVM at this feature size.
